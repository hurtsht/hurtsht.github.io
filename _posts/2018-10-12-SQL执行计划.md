## MySQL SQL 执行计划

#### explain

增加explain的mysql语句并不是不会执行查询，如果查询在from子句中包括子查询，那么mysql实际上会执行子查询，将其结果放在一个临时表中，然后完成外层查询优化。着意味着如果语句包含开销较大的子查询或使用临时表算法的视图，实际上会给服务器带来大量工作。



### id | select_type | table | type | possible_keys | key | key_len | ref | rows | Extra

#### table  表示哪张表

#### type展示连接使用的类型，最好到最差为system,const,eq_reg,ref,range,index和all

1. system ：const的特殊情况，表只有一行。

2. const：通过索引一次就找到了。

3. eq_ref：唯一性索引扫描。

4. ref：非唯一性索引扫描。

5. range：只检索给定范围的行，使用一个索引来选择行。

6. index : 另外一种形式的全表扫描，但是扫描顺序是按照索引的顺序扫描，先扫描索引，然后回表取数据

7. all : 全表扫描

   

#### possible_keys

查询涉及到的字段上存在索引，则该索引将被列出，但不一定被查询实际使用。

#### key

实际使用到的索引。

#### key_len

表示索引中使用的字节数，查询中使用的索引的长度（最大可能长度），并非实际使用长度，理论上长度越短越好。10.39.27.18

#### ref

显示索引的哪一列被使用了。

#### rows

根据表统计信息以及索引选用情况，估算出所需要读取的行数。

#### extra

额外信息
